Option A · Sort experience

The compact icon-only sort button needs a thoughtful menu to do its job. Here's the icon set, the picker layout, and how the trigger communicates its current state.

01 · Icon set

Glyphs for each sort mode

Each mode has a unique 14×14 glyph tuned for Option A's 24×24 button. Icons communicate what is being sorted by, not direction — direction is carried by the label ("Most recent" vs "Oldest") since it's clearer than two arrows.

Most recent
newest → oldest
Oldest first
oldest → newest
Duration
longest → shortest
Status
failed → running → ok

Why these glyphs

  • Most recent — clock with hand at 2 o'clock. Universal "time" signifier.
  • Oldest first — same clock, hand pointing back (8 o'clock) with subtle motion hash marks. Distinguishes from "Most recent" in the menu without being a different metaphor.
  • Duration — ascending bar-chart, reads as "lengths". Avoids confusion with the generic sort-arrows glyph that's on the trigger itself.
  • Status — three dots at decreasing opacities, the shared "status dot" vocabulary you already use in the run strip.

Menu anatomy

  • Header label — "Sort by" in mono caps, tiny. Orients the user in the half-second before they parse options.
  • Icon column — the glyphs from section 1, at 14px. Reinforces the icon vocabulary so users start to recognize = duration on sight.
  • Direction hint — "newest ↓" etc, mono, right-aligned, subtle. Makes the sort ordering explicit without requiring a separate direction toggle.
  • Divider — between the two time-based sorts and the metric-based sorts (duration / status). Not essential, but helps group them.
  • Selected state — accent background + check. No checkmark in the unselected rows (the opt-check slot is reserved; opacity 0 keeps alignment consistent).
03 · Trigger state

How the icon-only button communicates its current mode

The trigger is 24×24 — too small to fit a label. Three signals carry the state outward: (a) tooltip on hover, (b) accent dot when sort is non-default, (c) list content itself. For users who need a persistent label, offer the "labeled trigger" variant below as a Tweaks-style preference.

DEFAULT · Most recent
No indicator — default state
MODIFIED · Duration
Accent dot · button fill darkens
OPEN
Button pressed · menu open
LABELED VARIANT
For wider panels — shows active mode inline

Recommended behavior

  • Default sort = Most recent. No dot; no chrome change on the trigger. The icon-only button blends into the pill row.
  • Any other sort = accent dot + darker button fill. Users can see at a glance "this list is in an unusual order."
  • Hover → tooltip "Sort: {current mode}". Click → menu opens.
  • Auto-upgrade to labeled trigger when the panel is ≥400px wide AND a non-default sort is active. Gives precious vertical-scan info without costing space in narrow panels. Controlled by a @container query on the toolbar.
  • Keyboard: ↑/↓ to move selection, Enter to select, Esc to close. Same as your existing rerun scrim menu.
  • Persist the choice in state.executionsSort across sessions (you already do).
TL;DR

Shipping checklist

  • Add 4 glyphs to your icon sprite: sort-recent, sort-oldest, sort-duration, sort-status.
  • Reuse the existing .pl-sort-menu styles — extend with .menu-opt .opt-ico and .menu-opt .opt-dir slots.
  • Replace the current toggleExecutionsSort cycle-through with an anchored popover (use your existing rerun-scrim pattern).
  • Show accent dot on the trigger when state.executionsSort !== 'recent'.
  • Optional: container query to swap in the labeled trigger at ≥ 400px.
